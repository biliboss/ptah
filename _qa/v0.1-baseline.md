# v0.1 — baseline TTFA (Mac Air M4)

Data: 2026-06-03
Binary: `zig-out/bin/agent-tts` (Zig 0.16, ReleaseFast, 415KB, arm64 Mach-O)
Voz: `Luciana` (plain, sem Premium instalado)
Comando: `say -v Luciana -r <rate> <text>` via `std.process.spawn`

## Spawn latency (5 cold runs, rate 180, texto curto)

| Run | spawn (ms) |
|-----|-----------|
| 1   | 0.8       |
| 2   | 0.8       |
| 3   | 0.5       |
| 4   | 0.8       |
| 5   | 0.7       |

Média: **0.72ms**. Mediana: 0.8ms.

> Métrica = tempo entre process start e `std.process.spawn` retornar. **Não é TTFA real** — não captura o tempo entre `say` exec e primeiro sample audível. Real TTFA precisa `dtruss -t write` no `say` + captura de áudio (v0.2+).

## Rate sweep (Luciana, mesma frase, ~10 palavras)

| Rate (wpm) | Total (ms) |
|------------|------------|
| 180        | 3953       |
| 220        | 4103 (frase diferente) |
| 360        | 3862       |
| 450        | 3164       |
| 540        | 2651       |
| 600        | 2428       |

Redução de tempo total cai ~linear até 540, depois plateau. Acima de 600 inteligibilidade despenca (não medido formalmente, julgamento por ouvido).

## Default escolhido: 330 wpm

Decisão tomada por ouvido (Gabriel, 2026-06-03):

- 180 = lento demais pra agente
- 430 = inteligível mas seco/atropelado em frase longa
- 330 = sweet spot — humano, rápido, sustenta frase de ~80 palavras sem perder palavra

## v0.1 → v0.2 gaps

- TTFA real (audio sample detection) ainda não medido
- Pre-warm não existe — primeira chamada do dia paga cold do Speech Synthesis Manager
- Sem daemon = cada call paga fork+exec do `agent-tts` + fork+exec do `say`
- Sem fila = chamadas paralelas se sobrepõem (UX ruim)

Tudo isso vira meta de v0.2.
