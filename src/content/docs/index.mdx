## Why ptah

Your AI agents already write, build, deploy, and review. They should also **talk** — without stealing focus, without a Python venv, without paying ElevenLabs per token, without a shell-out from every tool.

```bash
# CLI
ptah "Build green in 12 seconds."         # Kokoro Dora default
ptah --voice pm_alex "Deploy concluído."  # alternate voice
ptah history --limit 10                   # persistent timeline
ptah pause; ptah resume; ptah replay 42

# MCP (Claude Code, Cursor, Cline) — 13 tools
claude mcp add ptah /opt/homebrew/bin/ptah mcp
# say · queue · skip · clear · voices · say_stream · pause · resume · replay · history
# synth_voice_test · voice_knob_search · tech_profile_search
```

## One KPI: time-to-first-audio

Everything is benchmarked against the gap between calling `ptah "text"` and the first audible sample.

| Scenario | Target | Measured |
|---|---|---|
| Warm CLI ack round-trip | < 5 ms | **0.2–0.4 ms** |
| Kokoro Dora warm synth (short) | < 300 ms | **~80-150 ms** |
| Long-input first audio (streaming, v1.2) | < 200 ms | **~50 ms** |
| MCP warm tool-call (persistent process) | < 10 ms | **~1–3 ms** |
| First call after boot (cold Kokoro) | < 3.5 s | model load ~3 s; warm ever after |

## Locked constraints

- **Platform**: macOS (primary). afplay native. Linux compile paths via espeak-ng + systemd.
- **Footprint**: single < 2 MB binary (`zig build`). Kokoro model (~310 MB) + voice packs opt-in download to `~/.cache/ptah/`.
- **Architecture**: CLI + daemon + MCP share the binary via UNIX socket protocol.
- **Pt-BR human cadence**: cardinals, abbreviations, and punctuation pauses preprocessed in 2–5 µs.
- **Offline by default**: no cloud, no telemetry, no quota. Voice models cached locally.
